# Agent Commerce 规范配置器与就绪检查器 (Agent Commerce Configurator & Readiness Checker) — 项目需求文档

本文件整理并记录了 **Agent Commerce 规范配置器与就绪检查器** 的核心功能、设计原则及项目架构要求。

---

## 1. 项目概述 (Project Overview)

本项目是一款旨在加速商家接入 **Agent Commerce 协议** 与 **x402 即时结算关口** 的一站式开发者工具平台。它允许人工智能代理（AI Agents）在无人工干预的情况下，自动识别计算节点、检索服务计费（Compute Credits）并进行去中心化支付结算。平台提供了**配置自动生成器**、**就绪检查工具（Readiness Checker）** 以及 **智能体交易沙盒模拟器**。

---

## 2. 核心功能需求 (Core Functional Requirements)

### 2.1 规范配置生成器 (Agent Specs Generator)
*   **多维输入生成**：用户输入网站 URL（例如 `https://yourstore.com`）后点击“生成配置”阶段。
*   **智能参数预测/生成**：
    *   在后台结合 URL 通过大模型或匹配逻辑，自动解析商家/节点名称，若非 AI 主题则自动在结尾修饰如 `"Yourstore Compute Node"`。
    *   将商家类别统一规划至 `AI Computing & APIs`。
    *   自动生成标准的 AI 算力计费配额包（`Starter Pack`、`Developer Pack`、`Pro Pack`）。
*   **配置文件三剑客**：
    *   `agent-commerce.json`：说明商店名称、合规货币、商品 Feed 链接以及支付服务提供商（x402）。
    *   `products.json`：算力额度包的具体定价（USDT/USD）、代号及库存就绪状态。
    *   `x402.json`：定义去中心化即时结算规则，支持配置 Base 等网络及商户加密收汇收款地址。
*   **手动覆盖模式（Override Schema）**：提供高级编辑区，允许开发者直接自定义修改商户名称、货币符号、算力产品 JSON 表格及商户接收端钱包地址，并能一键保存覆盖。

### 2.2 就绪检查器 (Readiness Checker)
*   **一键验证**：检测商户或算力节点的配置文件部署情况。
*   **链路诊断步骤**：
    1. 连通性测试：确认域名基础 HTTP 握手。
    2. 配置校验：读取并解析 `/.well-known/agent-commerce.json`。
    3. 特性比对：验证 `products.json` 中算力额度与结算协议格式。
    4. 结算层安全：检查安全支付协议 `x402.json` 的规范性。
    5. 综合评级：若完全就绪反馈。
*   **结果分流模拟**：支持测试“就绪成功路径”和“就绪失败路径（如 404 缺失、节点拒绝、非法地址）”，以帮助开发者了解在各种外部异常下系统的提示机制。

### 2.3 智能体交易沙盒模拟器 (Agent Sandbox Chat Simulator)
*   **实时会话驱动**：模拟一个真实的自主 AI Agent（如代号为 *HyperScale AI Crawler*）在检测到协议文件后的自动化决策和商品采购周期。
*   **卡片式结账交互**：在聊天气泡中内置交互式结算状态，当 Agent 下单算力包后，页面右侧或对话中同步高亮并流式输出 x402 结算哈希、支付区块以及付款地址（自动提取并渲染用户自定义的 `x402.merchant` 钱包）。

## 3. 视觉设计与交互体验规范 (Visual & UX Guidelines)

*   **视觉风格（Mood）**：前沿科技感、瑞士现代主义与极客暗色主题完美契合。使用深邃煤炭黑/幽蓝底色，辅以冷绿色与冷蓝色霓虹发光颗粒，构建富有未来主义格调的无波折质感。
*   **响应式布局**：
    *   桌面优先，保证在宽屏显示下采用多列 Bento 网格卡片（左侧数据校验及结果，右侧三叶规约与实时沙盒），无任何边界冗余。
    *   移动端平滑适配，针对触屏目标增加交互面积。
*   **极简精致滚动条**：
    *   全站元素采用细窄微扁平化的暗色高感滚动条，在不活跃时呈隐匿状态，悬停时柔和虚化变亮，杜绝原生滚动条破坏纯暗黑科技美感。
*   **纯文字字标与谦逊设计**：
    *   采用直观人性化的命名，导航条中主标题为 `Readiness Checker`。
    *   不带有任何未授权的“多余广告或模拟系统遥测”，所有日志均为与业务强相关的连通交互输出。

## 4. 技术栈支持 (Technical Specification)

*   **前端框架**：Next.js 15+ (App Router) 基于 React 19。
*   **动画过渡**：使用 `motion/react`（原 `framer-motion`）精雕打磨每个操作反馈与模态弹窗切换。
*   **图标系统**：统一从 `lucide-react` 导入标准线形组件。
*   **样式库**：由 Tailwind CSS 全权驱动。
*   **状态设计**：防抖计算器视窗大小变更，使用 React 内置 State 设计防范闭环重绘。
